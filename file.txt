Философия тестирования:
-- постановка задачи теста (! самое главное) 
-- проектирование, написание тестов
-- тестирование тестов
-- выполнения тестов
-- изучение результатов тестирования
--------------------------------------------
-- находиться в движении
-- непрерывно развиваться
-- всегда уметь задавать неудобные вопросы
--------------------------------------------
Концепции тестирования:
-- 100% проверка ПО. 99%. 
Найти как можно больше приоритетных багов.
Пропустить как можно меньше менее приоритетных.
-- критерий эффективности тестирования 
- это кол-во багов найденныйх до релиза
---------------------------------------------
Пострелизные баги
Релиз - передача ПО пользователю.
1й релиз = 50
2й релиз = 200

Types of testing / классификация
По знанию внутренней системы:
- black box testing -- 
область незнания тестируемой части бэк-энда.
применение паттерна.
- white box testing --
знание устройства и логики тестируемой части БЭ
- grey box testing --
вид тестирования, сочетающий элементы Ч и Б ящиков.
не используются паттерны поведения пользователя.
===========
По объекту:
- functional testing --
проверка функционала ПО
- load/performance/stress testing --
проверка поведения продукта 
при одновременном наплыве множества пользователей
- ui testing --
проверка элементов интерфейса пользователя
- localization testing --
проверка множества аспектов, связанных с адаптацией продукта
для пользователей из других стран.
===============
- security testing --
проверка множества аспектов, суть которых заключается в
усложнении условий для кражи
- usability testing --
вид тестирования, позваляющий объектно оценить опыт пользователя
- compatibility testing --
похволяет проверить возаимодействие между "сайтом" и ПО
================================
По субъекту тестирования
Альфа-тестер --
сотрудник который про/непрофессионально проводит тестирование
Бета-тестер --
тестировщик, не являющийся сотрудником
========
По времени проведения тестирования:
Альфа-тестирование --
до передачи
Бета--
после передачи
====
Виды тестирования провожящиеся между альфа и бета тестированием:
- smoke test -- тест приемки --
быстрый, поверхностный, не ацентирющий на деталях тест
минимальный набор тестов
- new feature testing -- тестирование новых функциональностей

- regression testing -- регрессивное тестирование
тестирования абсолютно всего функционала
- acceptance test -- тест сдачи
тест перед сдачей
=====================
По критерию "позитивности" сценариев:
- positive testing --
тестирование предпологающее правильное нормальное использование
и работу системы
- negative testing --
сценарий проверяющий ситуацию связанную с потенциальной ошибкий
и потенциальным дефектом
